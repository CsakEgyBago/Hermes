The new expense reporting tool rolled out to all departments this month. IT reported a smooth migration with no downtime. A rollback plan was prepared in case issues surfaced after go-live. The help desk saw a small spike in password reset requests unrelated to the change. Access permissions were audited before the cutover.

IT's internal notes credit the smooth migration to an extended testing window with a pilot group from three departments over the preceding month, which surfaced and resolved several minor configuration issues well before the full rollout. The rollback plan was never invoked, though IT considers having it ready a key reason the team felt comfortable proceeding with a single cutover rather than a slower phased rollout. The access permission audit found a small number of stale accounts that had lingered from a reorganization earlier in the year, which were cleaned up as part of the same pass.
