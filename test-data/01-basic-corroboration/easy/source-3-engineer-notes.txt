The Q3 server outage was caused by a memory leak in the caching layer. A hotfix was rolled out that added a maximum size limit to the cache. Root cause analysis is still ongoing for a secondary latency spike. Heap snapshots confirmed the growth pattern matched the suspected cache object. The fix has since been backported to the staging environment for regression testing. A ticket was filed to add automated alerts for unbounded memory growth. Pairing with the infrastructure team helped narrow down the affected service within twenty minutes.
